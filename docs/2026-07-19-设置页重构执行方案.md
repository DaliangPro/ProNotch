# ProNotch 2.0 设置页七区重构执行方案

- 日期：2026-07-19（v2：两项决策已拍板；经系统自查修订，修正了 v1 的两处会成真 BUG 的遗漏，见第六节）
- 状态：已拍板，待大梁老师下开工指令
- 前置：2.0 第一轮（模块真停机 + Agent 联动）已提交（e2cd123 / 4b29d79 / c06e494 / c4e9bac）

## 一、背景与目标

2.0 核心原则：**功能的开启关闭必须真正节约系统资源，而不是只在视觉上关掉**。第一轮已把各模块的「关=停资源」做实；本轮（第二轮）解决两个遗留问题：

1. **设置页信息架构混乱**：现在只有 5 个分区，「通用」是个杂物抽屉——剪贴板、天气预警、刘海槽位全挤在里面，与功能模块对不上号。
2. **刘海面板页面与内容脱节**：取消全部 Agent 勾选后，「额度」「Agent」两个页签还挂在面板上，点进去是空壳。

目标：设置页重排为**七区**（通用 / 刘海面板 / Agent / 组件 / 超级截图 / AI 闪问 / 关于），刘海页面**跟随内容自动显隐**（已拍板：不设独立页面开关），完成后升版本号 **2.0.0**。

## 二、范围边界（本轮不做什么）

- **只搬家不改行为**：所有设置项的功能、存储 key、通知链一律不动，纯 UI 重排。
- **不动数据层**：真停机联动第一轮已完成，本轮不碰任何 Store 的资源逻辑。
- **不加页面开关**：页面显隐完全由内容推导，不新增任何持久化状态。
- 不含 README/docs 截图重拍的内容创作（列为发版前置步骤，另行执行）。
- Kimi 会话榜 Top 5、ZCode 升级正式家：不在本轮。

## 三、现状盘点（代码实况，2026-07-19）

### 刘海面板现有 5 页（NotchViewModel.Tab）

启动台 launcher / 闪问 chat / 额度 usage / Agent agent / 组件 widgets（内存卡＋天气卡）。页签顺序 `tabOrder` 可拖动并持久化，排第一的是默认页；有中文旧值兼容映射（legacyNames）。

### 关键耦合点（重构时必须核对）

- `SettingsStore.pendingSection`：外部入口跳转设置分区，置值处全部用 `SettingsView.Section.xxx.rawValue` 类型安全引用（ChatView:113、ExpandedContentView:288、UsageView:48）——枚举改动编译期自动暴露，**无字符串裸写**。
- Agent 区的 `probeResults` 扫描挂在该区显示时机上，搬迁时 onAppear 挂点要跟着走。
- 主菜单「Agent 额度」勾选与设置页共用 `showUsageInMenuBar`（第一轮已统一）。

## 五、页面自动显隐设计

### 判定口径（每页一条规则，全部现算、零新增持久化）

| 页 | 显示条件 | 依据 |
|---|---|---|
| 启动台 | 常显 | 面板的兜底页，保证永远至少一页可见 |
| 闪问 | 常显 | 未配置时页内自带配置引导——隐藏会切断新用户的发现路径 |
| 组件 | 常显 | 内存卡零配置永远有数据 |
| 额度 usage | `enabledAgents ∩ supportsQuota ≠ ∅` | 一家没勾就没有任何额度可看 |
| Agent agent | `enabledAgents ∩ supportsSessions ≠ ∅` | 只勾了 Grok（无会话能力）时监控台必空 |

### 技术落点

- `NotchViewModel` 新增计算属性 `visibleTabs: [Tab]`＝`tabOrder.filter(按上表)`；勾选集从静态 `AgentKind.enabledSet()` 读（与数据层同口径，不引入对 SettingsStore 的新依赖），订阅 `ProNotchAgentSelectionChanged` 触发 `objectWillChange` 并做 activeTab 兜底。
- `ExpandedContentView` 页签行 ForEach 改用 `visibleTabs`；`tabOrder` 持久化数组**不删项**——被隐页的位置保留，勾回 Agent 后原位恢复。
- 兜底规则：`activeTab` 正指向被隐页时，切到 `visibleTabs.first`（启动台常显保证非空）；面板锁定态同样适用。
- 资源面零新增工作：usage/agent 页隐藏时数据层早已按 `enabledAgents` 停止（M1 已做），本轮只是让 UI 不再展示空壳。

### activeTab 全部写入点普查（自查补充：v1 只列了渲染层与 init 兜底，switchTab 与调试循环是遗漏）

| 写入点 | 目标页 | 是否需改 |
|---|---|---|
| `NotchViewModel` init（`order.first ?? .launcher`） | 持久化首位 | **要改**：首位被隐时取第一个可见页 |
| **`NotchViewModel.switchTab(by:)`（触控板横滑/Shift+滚轮循环切页）** | tabOrder 循环 | **要改**：改在 `visibleTabs` 上取模循环，否则横滑能把页签栏里都不存在的隐藏页滑出来（v1 真 BUG） |
| AppDelegate ×2（闪问快捷键/入口 → `.chat`） | 闪问 | 常显，零改动 |
| NotchContainerView（天气大卡点击 → `.widgets`） | 组件 | 常显，零改动 |
| NotchWindowController `debugNextTab`（调试循环） | allCases 循环 | 顺手改 visibleTabs（仅调试路径） |

### 拖拽排序的索引空间问题（自查补充：v1 的第二处真 BUG）

现状拖拽实现（ExpandedContentView 的 TabButton）用固定 50pt 步距把**视觉位移直接换算成 `tabOrder` 索引**再 `move`。页签行改渲染 `visibleTabs` 后，视觉索引空间与 `tabOrder` 索引空间在有隐藏页时**不再一致**——照 v1 执行，拖动跨过隐藏页槽位时换算错位、落点和动画全乱。

对策：拖拽的 current/target 全部改算在 `visibleTabs` 索引空间；松手后用**稳定合并**写回——可见项按新顺序依次填回 `tabOrder` 中原本属于可见项的槽位，隐藏项槽位原地不动。合并做成纯函数并配单测（用例必须含「隐藏页夹在两个被交换的可见页中间」）。

### 边界情形清单（点验时逐条过）

1. 停在额度页时取消最后一家勾选 → 页面即时消失并自动落到可见页，无空白帧。
2. 重新勾选 → 页签回到 tabOrder 中的原位置（不是排到末尾）。
3. 默认页（tabOrder 首位）被隐 → 展开落在第一个可见页。
4. 拖动排序时被隐页不出现、不可拖，但其存值位置不被挤掉。
5. 只勾 Grok → 额度页在、Agent 页不在（能力矩阵驱动，不是一刀切）。

## 六、已拍板决策与自查结论

### 已拍板（2026-07-19 大梁老师确认，按推荐执行）

1. **「软件更新」行搬到通用区**：对齐 macOS 设置习惯；关于区保留版本号展示。
2. **快捷键跟着功能走**：截图/剪贴板/闪问三个快捷键留在各自功能区，不新设「快捷键」区。

### 系统自查结论（v1 → v2 修订记录）

对照代码逐点核查后，v1 有两处照做会出真 BUG 的遗漏、四处需要加固的细节，已全部修入正文：

1. **横滑切页漏改**（真 BUG）：`switchTab(by:)` 在 tabOrder 上循环，会把隐藏页滑出来 → 第五节普查表。
2. **拖拽排序索引空间错位**（真 BUG）：视觉位移按固定步距换算 tabOrder 索引，有隐藏页时错位 → 第五节稳定合并对策。
3. **钥匙串弹框陷阱**：超级截图区的 `.onAppear { translateKey = 钥匙串读取 }` 只能挂在该区内容上，搬迁时若误上移到 body 级，每次开设置窗都弹钥匙串授权框 → S1 验证标准。
4. **Agent 区三个修饰符成组**：`probeResults` 扫描的 `.onAppear` 与 glowEnabled/enabledAgents 两个 `.onChange` 都挂在区内容上，必须随区整体走 → S1 验证标准。
5. **搬迁单位是「行」不是「卡」**：通用区第一张卡是三行合一（开机自启/全屏隐藏/剪贴板记录），拆去三个区时 CardDivider 结构要重排，机械搬卡会搬错 → S1 验证标准。
6. **测试缺口**：v1 完成定义只要求存量 46 项测试通过，S3 的新逻辑无覆盖 → S3 新增三组纯函数单测（见实施步骤）。

自查同时确认无风险的点：设置窗 660×540 固定尺寸下侧栏 7 项放得下（约 217pt < 可用高度）；侧栏纯文字无需定图标；`pendingSection` 三处入口全是类型安全引用；`-snapshotPanel` 截图渲染实例的 enabledSet() 兜底全开、四页全显不受显隐影响；额度页可见性按「勾选」而非「已安装」判定，与设置页幽灵行口径一致（勾着但已卸载的家保留关闭入口，页面同理保留）。
